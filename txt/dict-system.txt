На данный момент я имею собранную базу из двух источников прогрнанную
через бота получения перевода и бота получения произношения.

К сожалению я поторопился и не сделал контрольных точек на всех этапах

В будущем для удобства работы со словарями можно сделать такую систему

Отдельно хранить сырой источник каждой базы, наприммер
word_os (opensubtitles, 15000)
word_wd (word frequentcy free 5000)

которые буду наследоваться от общей таблицы, чтоб быть совместимыми с ботами
* word
* frequency
* translation (string, null)
* pronounce (bool, null)

Таких таблиц может стаить и больше, каждую из них можно прогнать через бота

Бот перевода будет обрабатывать слова с translation null
и запрашивать их у гугла (инъекция в интерфейс переводчика)
Но предварительно будет смотреть в кеш таблице

Бот произношения будет смотреть записи с pronounce null
и пытаться их скачать но сначала проверять в кеш таблице
скадывать слова будет в папке /named-pronounce/
а в поле только ставить true / false
true - произношение есть в папке
false - произношения нет ни в папке ни у гугла
null - произношение не запрашивалось

Соответственно будет две доп таблицы, кеш произношений и кеш перевода

Заключительная фишка этой системы - команда компилирования общего словаря
Будет брать указанные словари и собирать в один общий, уже непосредственно
подключая файлы произношений с их переименованием
Сортировать будет по полю frequency в каждой таблице, используя при 
переносе данных индивидуальный для каждой базы мультпликатор
